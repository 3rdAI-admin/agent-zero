# Directory Structure

This document describes the professional directory structure of Agent Zero.

## Root Directory

```
AgentZ/
├── README.md                 # Main project README
├── docker-compose.yml        # Docker Compose configuration
├── Dockerfile                # Docker build file
├── DockerfileLocal           # Local Docker build file
├── agent.py                  # Core agent entry point
├── run_ui.py                 # Web UI server
├── run_tunnel.py             # Tunnel server
├── initialize.py             # Framework initialization
├── prepare.py                # Environment preparation
├── preload.py                # Pre-initialization
├── models.py                 # Model configurations
└── .env                      # Environment variables (user config)
```

## Core Directories

### `python/`
Core Python codebase:
- `api/` - API endpoints
- `extensions/` - Modular extensions
- `helpers/` - Utility functions
- `tools/` - Tool implementations

### `webui/`
Web interface:
- `js/` - JavaScript modules
- `css/` - Stylesheets
- `html/` - HTML templates
- `public/` - Static assets

### `prompts/`
System and tool prompts for agents

### `knowledge/`
Knowledge base storage

### `agents/`
Agent definitions and configurations

### `instruments/`
Custom scripts and tools

## Organized Directories

### `scripts/`
All executable scripts organized by purpose:
- `setup/` - Installation and configuration
- `testing/` - Test and validation scripts
- `maintenance/` - Maintenance and fix scripts
- `docker/` - Docker-related scripts

### `docs/`
Documentation organized by category:
- `guides/` - Setup guides and how-tos
- `integration/` - Integration documentation
- `troubleshooting/` - Troubleshooting guides
- `archive/` - Historical/obsolete docs
- `res/` - Documentation assets

### `config/`
Configuration files:
- `examples/` - Example configurations

### `docker/`
Docker-related files:
- `base/` - Base image files
- `run/` - Runtime image files

### `tests/`
Test files and test utilities

## Data Directories (Volume Mounts)

These directories are mounted as Docker volumes:
- `memory/` - Agent memory storage
- `logs/` - Log files
- `tmp/` - Temporary files
- `claude-config/` - Claude Code configuration
- `claude-credentials/` - Claude Code credentials

## Standards

- **Scripts**: All in `scripts/` organized by purpose
- **Documentation**: All in `docs/` organized by category
- **Configuration**: Examples in `config/examples/`
- **Tests**: All in `tests/` directory
- **Root**: Only core files and entry points
