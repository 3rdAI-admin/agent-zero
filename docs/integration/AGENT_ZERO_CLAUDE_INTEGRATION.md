# Agent Zero + Claude Code Integration Guide

## Overview

Agent Zero can use Claude Code through its `code_execution_tool`, which executes terminal commands. Since Claude Code is installed at `/usr/local/bin/claude` (and `/usr/local/bin/claude-pro` for OAuth), Agent Zero can invoke it directly.

## How Agent Zero Uses Claude Code

### Method 1: Direct Terminal Execution (Recommended)

Agent Zero can call Claude Code using the `code_execution_tool` with `runtime: "terminal"`:

**Example Agent Zero Tool Call:**
```json
{
    "thoughts": [
        "I need to use Claude Code to help with this coding task",
        "I'll invoke Claude Code via terminal"
    ],
    "headline": "Using Claude Code for assistance",
    "tool_name": "code_execution_tool",
    "tool_args": {
        "runtime": "terminal",
        "session": 0,
        "code": "claude-pro 'Write a Python function to calculate fibonacci numbers'"
    }
}
```

### Method 2: Interactive Session

For longer, interactive sessions with Claude Code:

```json
{
    "tool_name": "code_execution_tool",
    "tool_args": {
        "runtime": "terminal",
        "session": 0,
        "code": "claude-pro"
    }
}
```

Then use the `input_tool` to send follow-up messages:

```json
{
    "tool_name": "input_tool",
    "tool_args": {
        "session": 0,
        "keyboard": "Explain this code: def fib(n): return n if n < 2 else fib(n-1) + fib(n-2)"
    }
}
```

## Available Claude Commands

### 1. `claude-pro` (OAuth - Recommended)
- Uses your Claude Pro subscription (no API costs)
- OAuth authentication is already configured
- **Usage**: `claude-pro "your question or task"`

### 2. `claude` (API Key - Fallback)
- Uses API key authentication (costs per use)
- Falls back to API if OAuth fails
- **Usage**: `claude "your question or task"`

## Important Notes

### Authentication
- **OAuth is configured**: The `claude-config` directory is volume-mounted, so OAuth tokens persist across container restarts
- **Use `claude-pro`**: Always use `claude-pro` to leverage your Pro subscription and avoid API costs
- **PATH is set**: Both `/usr/local/bin/claude` and `/usr/local/bin/claude-pro` are in PATH

### Interactive Sessions
- Claude Code is an interactive tool - it may prompt for input
- Agent Zero's `code_execution_tool` can handle interactive prompts using the `input_tool`
- For long-running conversations, use separate terminal sessions (different `session` numbers)

### Output Handling
- Claude Code output will be captured by Agent Zero's `code_execution_tool`
- The output will appear in Agent Zero's response and logs
- Large outputs may be truncated - check Agent Zero's logs for full content

## Troubleshooting

### Claude Code Not Found
If Agent Zero reports `claude-pro: command not found`:
- Verify installation: `docker exec agent-zero which claude-pro`
- Check PATH: `docker exec agent-zero echo $PATH`
- Reinstall if needed: `docker exec agent-zero bash -c 'curl -fsSL https://claude.ai/install.sh | bash'`

### OAuth Issues
If OAuth authentication fails:
- Check config persistence: `docker exec agent-zero ls -la /root/.config/claude-code/`
- Re-authenticate via VNC if needed (see `CLAUDE_CODE_AUTH.md`)

### Interactive Prompts
If Claude Code prompts for input and Agent Zero doesn't respond:
- Use `input_tool` to send keyboard input to the session
- Or restart the session with `runtime: "reset"` and try again

## Best Practices

1. **Use `claude-pro`** for all Claude Code invocations to avoid API costs
2. **Be specific** in prompts to Claude Code - include context and requirements
3. **Use separate sessions** for different Claude Code conversations to avoid confusion
4. **Combine with other tools** - Agent Zero can use Claude Code's output with `code_execution_tool` to execute the generated code
5. **Monitor output** - Check Agent Zero's logs to see Claude Code's full responses

This integration allows Agent Zero to leverage Claude Code's AI-powered coding assistance as part of its autonomous task execution capabilities.
